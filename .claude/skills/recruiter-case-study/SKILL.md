# Skill: Recruiter Case Study

Transforme kv-web-starter en argument de portfolio convaincant pour un recruteur ou client.

## Procedure

### 1. Identifier les points forts techniques

Pour kv-web-starter, les arguments cles sont:
- Architecture monorepo pnpm workspace
- Next.js App Router + TypeScript strict
- Design system multi-theme avec tokens CSS
- Auth JWT complet (Auth.js v5)
- Billing Stripe avec webhooks
- API REST avec cles + rate limiting
- Tests: Vitest (unit) + Playwright (e2e)
- Multi-langue (next-intl EN/FR)
- Accessibilite et mobile-first

### 2. Structure du case study

```
## Contexte
Pourquoi ce projet existe, quel probleme il resout.

## Architecture
Choix techniques cles et pourquoi ces choix.

## Defis resolus
3-5 problemes concrets et les solutions appliquees.

## Demo
Liens vers les pages demo par type de projet.

## Ce que ca prouve
Competences demonstrees par ce projet.
```

### 3. Adapter selon la cible

- Recruteur tech: insister sur la profondeur de la stack, les tests, la qualite du code
- Client business: insister sur la vitesse de livraison, les features, la demo
- Senior dev: insister sur les decisions d'architecture, les trade-offs, ce qui est omis intentionnellement

### 4. Les demos pages comme preuve

Pointer vers `/demo/*` pour montrer:
- La polyvalence (9 identites visuelles differentes)
- La qualite du design (themes professionnels)
- La logique produit (pas que du CSS)

### 5. Phrases a eviter

- "boilerplate" (dire "base professionnelle" ou "starter kit enterprise")
- "simple" (dire "epure" ou "minimaliste")
- "j'ai juste..." (ne pas diminuer le travail)
